tl;dr

This is a virtual lab for nuclear engineering undergrads. I'm responsible for all design, artwork, and (nearly all) programming; I built the lab in Unity and created the visuals in Blender, Marmoset Toolbag, Substance Painter, and Affinity Designer.

Purdue University Prof. Stylianos Chatzidakis led this project to increase access to curriculum-necessary lab equipment.

Completed late 2023

These appliances are close matches to their real counterparts, so a student should be able to use the real equipment after using this. All of the controls (knobs, dials, switches, etc) are fully functional.

The lab runs in a browser and is optimized for low-spec devices such as Chromebooks and tablets. There are max 25,000 triangles rendered across 22 draw events. The lab's load size is just under 50mb.

Some subtle details in the models:

  • Texel density is lower on obscured or unseen faces.
  • Lighting is baked directly into unlit equipment materials using Blender / Cycles since each only appears once.
  • I created specific reflection cubemaps for many non-static buttons and dials to prevent incorrect specular highlights.
  • Some movable objects drop soft-shadow decals to make their position easier to read.

programming

Built in Unity, I'm responsible for most of this project's scripting (UI, 3D equipment controls, object manipulation, cable management, camera framing, data graphing, etc, etc). Like the 3D artwork, everything is profiled and optimized for low-spec devices.


Signal processing and equipment simulation:

Each tool simulates signal processing on voltage curve functions. With the help of Prof. Chatzidakis and his team, I modeled the equipment behavior in C#.

The lab targets 100,000 data samples per second, each a voltage curve signal processed by the equipment. Being a web app, I'm limited to a single CPU core and can't use Unity's Burst compiler. Even so, low-spec devices run the simulation at full speed.

The data histogram and oscilloscope line graph displays are shaders. Some equipment will write incoming data to textures that these shaders display as a graph.


Cable visuals and performance:

For cable physics I use Obi Rope. My Obi settings are tuned for a decent balance between visual quality and performance.

Cable mesh updates are expensive; time increases linearly with vertex count. I use flat ribbon meshes with cylindrical normals to minimize vert count.

I randomize cable colors so users can keep track of what is connected to what. All cables use the same material with different vertex colors.

The cables become semi-transparent via dithering when they may obscure important elements.


User experience in 3D:

The user shouldn't lose draggables behind other objects or off screen, so the user cannot drop objects behind UI elements, outside of the camera bounds, or while the camera is moving.

Held objects position using rigidbody-casts so that they don't visually clip into their surroundings.

I need to support arbitrary aspect ratios, so I wrote an editor tool for creating camera 'frames.' These keep important objects visible no matter the display aspect.

Controls must be large enough for easy interaction, so I position frames to keep controls at sane screen-sizes. I also keep related frames and controls close together.
